scrapy框架

scrapy框架是Python封装了一些高频爬虫功能的框架，具体介绍如下

scrapy框架（封装的高频功能）
1.高性能的持久化存储
2.异步的数据下载
3.高性能的数据解析
4.分布式

下载步骤
1.pip install wheel
2.下载twisted,下载地址为http://www.lfd.uci.edu/~gohlke/pyhonlibs/#twisted
3.安装twisted:pip install Twisted-17.1.0-cp36-cp36m-win_amd64.whl
4.pip install pywin32
5.pip install scrapy

一、基本操作（在pycharm下方的终端页输入)

(1)创建一个工程
scrapy startproject projectName
(2)cd到projectName文件的子目录(有spiders文件夹的上级目录)
(3)在工程的spiders子目录中创建一个爬虫文件
scrapy genspider spiderName www.xxx.com
(4)将spiders同目录下的ROBOTSTXT_OBEY 属性改为False(不遵从robot协议)
并增添一行 LOG_LEVEL = ‘ERROR’ ,只打印错误的日志信息，且可以指定其中的USER—AGENT
(5)执行工程
scrapy crawl spiderName
(6)数据解析
基于xpath进行解析：
1.xpath返回的是列表，但是列表元素一定是Selector类型的对象
2.某个元素调用extract()方法，可以将Selector对象中data参数存储的字符串提取出来
3.某个列表调用extract()方法，可以将列表中所有Selector对象中data参数存储的字符串提取出来
4.随后调用join():连接字符串数组，将列表中的字符串都连接成一个新的字符串
5.变量没有提示可调用方法，可以先查看变量的类型，并在其后边注释类型信息，注释后即可调用
(7)持久化存储
基于终端指令：只可以将parse方法的返回值存储到本地的文本文件中
scrapy crawl spiderName -o ./filename 将执行工程的返回数据存入当前目录的文件中
且文件类型只能为’json’, ‘jsonlines’, ‘jsonl’, ‘jl’, ‘csv’, ‘xml’, ‘marshal’, ‘pickle’
基于管道：
1.数据解析
2.在item类中定义相关的属性
3.将解析到的数据封装存储到item类型的实例化对象，将item类型的实例化对象提交给管道 yield item
4.在管道类pipelines的process_item方法中，将接收到的item对象中存储的数据，进行持久化存储操作
-继承只执行一次的打开文件和关闭文件方法
-在process_item方法中向打开的文件写入数据
5.在配置文件中，开启管道（解除以下代码的注释）
ITEM_PIPELINES = {
“mn_pic.pipelines.MnPicPipeline”: 300,
}

二、基于Spider的全站数据爬取

将网站某版块下的全部页码对应的页面数据进行爬取
方法：进行手动请求的发送 yield scrapy.Request(url=要发送请求的url,callback=回调的方法(如self.parse)) callback专门用作于数据解析

三、五大核心组件

（1）引擎（Scrapy）
用来处理整个系统的数据流处理，触发事务（框架核心）
（2）调度器（Scheduler）
用来接收引擎发过来的请求，压入队列中，并在引擎再次请求的时候返回。
（3）下载器（Downloader）
用于下载网页内容，并将网页内容返回给蜘蛛（Scrapy下载器是建立在twisted这个高效的异步模型上的）
（4）爬虫（Spider）
从特定网页提取自己需要的信息，即所谓的实体(Item)
（5）项目管道（Pipeline）
负责处理爬虫从网页中抽取的实体，主要的功能是持久化存储实体中的有效信息

四、请求传参 案例nm_pic

  • 使用场景: 如果爬取解析的数据不在同一张页面中（有详情页）。（深度爬取）
  • 实现方法:
    (1)在主文件中实现一个新的方法 def parse_detail(self,response)
    (2)将parse中获取的url通过手动请求的发送将url发送到新的方法parse_detail中
    yield scrapy.Request(url=new_url,callback=parse_detail)
    (3)在parse方法中实例化item对象，并将item对象传给parse_detail方法中（即请求传参）,在上行代码中添加meta参数
    yield scrapy.Request(url=new_url,callback=parse_detail,meta={‘item’: item})
    (4)回调方法parse_detail通过response.meta[‘item’]来接收item
    item = response.meta[‘item’]
    (5)将回调方法中解析的数据存入接收到的item中，并将item提交至管道,在setting中开启管道
    yield item

六、中间件 middlewares 案例wypic

作用: 批量拦截到整个工程中所有的请求和响应
拦截请求:
(1)UA伪装(单独、部分伪装，不像setting中是全局伪装)
在下载中间件ZzpicDownloaderMiddleware的process_request方法中实现，代码如下:
#创建UA伪装列表
user_agent_list = [
‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/114.0.0.0 Safari/537.36’,
‘Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/114.0.0.0 Safari/537.36 Edg/114.0.1823.82’,
‘Mozilla/5.0 (Windows NT 10.0; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/69.0.3947.100 Safari/537.36’,
‘Mozilla/5.0 (Windows NT 10.0; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/108.0.5359.125 Safari/537.36 QIHU 360SE’]
#拦截请求
def process_request(self, request, spider):
#UA伪装,通过random函数从user_agent_list中随机选择UA
request.headers[‘User-Agent’] = random.choice(self.user_agent_list)
return None
(2)代理IP
在下载中间件ZzpicDownloaderMiddleware的process_exception方法中实现，代码如下:
#创建ip列表
proxy_http = [
‘153.180.102.104:80’,
‘195.200.131.189:56055’
]
proxy_https = [
‘120.83.49.90:9000’,
‘95.189.112.214:35508’
]
#拦截发生异常的响应（ip异常访问失败）
def process_exception(self, request, exception, spider):
#ip代理,修改ip使请求可以正常访问网址,调用request.meta中的proxy（代理）参数
if request.url.split[‘:’][0] == ‘http’: # 如果拦截到的是http请求，则从http列表中获取ip
request.meta[‘proxy’] = ‘http://‘+random.choice(self.proxy_http)
else: # 是https请求，则从https列表获取ip
request.meta[‘proxy’] = ‘https://‘+random.choice(self.proxy_https)
拦截响应:
(1)篡改响应数据、对象（获取网页动态加载的数据）
拦截响应（通过中间件拦截响应对象并篡改，随后将新的请求对象返回)
本案例是结合selenium来将响应对象改为可获取动态数据的selenium对象，代码如下:
def process_response(self, request, response, spider):
driver = spider.driver #定义driver获取了在爬虫类中定义的浏览器对象
# 挑选出指定的响应对象进行篡改 通过url指定request,通过request指定response
if request.url in spider.model_urls: # 如果请求的url是在爬虫文件中的model_url的url
driver.get(request.url) # 通过selenium的浏览器对象对该url重新发起请求
page_text = driver.page_source # 获取浏览器当前页面源码的数据
new_response = HtmlResponse(url=request.url,body=page_text,encoding=’utf-8’,request=request)
return new_response #返回新的响应对象
else:
return response #不满足则返回原来的响应对象

七、CrawlSpider 案例sunPro

是Spider的一个子类，专用于全站数据爬取的方式
使用方法:
(1)创建一个工程 scrapy startproject projectName
(2)cd 工程名
(3)创建爬虫文件（基于CrawlSpider） scrapy genspider -t crawl spiderName www.xxx.com
规则解析器，从左至右含义如下
1.链接提取器:利用正则表达式提取链接,/d为匹配一个数字，/d+为匹配多个数字
2.callback回调，将链接提取器提取到的链接传给回调函数进行下一步操作
3.follow改为true，可以将链接提取器继续作用到链接提取器所提取到的链接，即可以提取所有页的页码
rules = (Rule(LinkExtractor(allow=r”index_\d+”), callback=”parse_item”, follow=True),)

八、增量式爬虫
