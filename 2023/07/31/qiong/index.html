c++基础编程

const 修饰

const在谁前面谁就不可以更改，举例int *p,*p是值，p是地址，说明如下
当const修饰指针时，指针指向的内容不可被修改（常量）,地址可以被修改
当const修饰常量时，指针指向的值可以更改，指针的指向不可以更改
当const同时修饰常量和指针时，两者都不能更改

static 修饰

全局变量修饰：当一个源程序由多个源文件组成时，static修饰的全局变量只能在自己的源文件里使用
局部变量修饰：当在函数中定义的变量以static修饰时，该变量的地址是静态的，它只会被初始化一次

函数的声明

程序编译是自上而下的，若函数定义在main函数后边，则编译时会找不到该函数而出错。而函数提前声明后，声明的函数可以写在main函数的后边。

函数的分文件编写

1.创建后缀名为.h的头文件
2.创建后缀名为.cpp的源文件
3.在头文件中写函数的声明
4.在源文件中写函数的定义 (源文件中导入自定义的头文件需要加” “)
vscode不能直接按以上方法，携带源文件的cpp文件不会被编译，所以需要在设置中添加路径才行。

结构体嵌套

被包含的结构体需要在之前定义包含它的结构体之前先定义。

c++核心编程

内存分区模型

c++程序在执行时，将内存大方向划分为4个区域
(1)代码区：存放函数体的所有的二进制代码(无注释)，操作系统进行管理。（共享,只读）
(2)全局区：存放全局变量和静态变量以及常量

//全局变量和static定义的局部变量都在全局区，const修饰的局部常量仍在不在全局区
int a;
int main(){
    int b;
    static int c;   //静态定义变量
    const int d = 1;//定义的常量存放在常量区
    cout<<(int)&a<<endl;
    cout<<(int)&b<<endl;
    cout<<(int)&c<<endl;
    cout<<(int)&d<<endl;//可得到a和c在全局区，而b和d不在全局区
}

(3)栈区：由编译器自动分配释放，存放函数的参数值，局部变量等(不要返回局部变量的地址，会变)
(4)堆区：由程序员分配和释放，若程序员不释放，程序结束时由操作系统回收

//利用new关键字，可以将数据开辟到堆空间，由程序员管理
int *a = new int(10);//new创建一个int类型的数据，并返回其的地址赋予指针
cout<<*a<<endl;
delete a;//使用delete关键字删除堆区中的变量
//cout<<*a<<endl;//此时会报错
